# Agent Skills

本仓库存放可安装到 Codex 等 coding agent 的 skills。每个 skill 都放在 `skills/<skill-name>/` 下，并以 `SKILL.md` 作为入口；脚本、数据池、参考资料等资源随 skill 自包含维护。

推荐在 `GPT-5.5` 且 reasoning effort 设为 `high` 的 Codex 会话中使用本仓库的创作型 skill。该配置更适合处理长设定、多约束、中文文化语境和命名风格一致性。

## 当前 Skill

- `xiuzhen-art-generator`：修真/仙侠设定生成器。用于随机或按条件生成受控提示词，并据此创作法术、功法、秘籍、法宝、神通、百艺、剑法、修真职业或相关体系。

该 skill 的核心模型是：

```text
法门 = 天位(动力因：为何成力/何因增强/何时耗尽/失控何状/如何度量) x 地位(媒介/承载方式) x 人位(目的/作用)
```

## 安装

推荐使用 `skills` CLI 从 GitHub 链接安装。下面的命令会从 `https://github.com/moyojo/skills` 拉取 skill，不依赖本地仓库路径。

先查看仓库中可安装的 skills：

```bash
npx skills add https://github.com/moyojo/skills --list
```

安装到当前项目，供 Codex 在该项目内使用：

```bash
npx skills add https://github.com/moyojo/skills --skill xiuzhen-art-generator --agent codex --yes
```

安装为全局 Codex skill，供所有项目使用：

```bash
npx skills add https://github.com/moyojo/skills --skill xiuzhen-art-generator --agent codex --global --yes
```

查看已安装 skills：

```bash
npx skills list --agent codex
```

更新时重新运行同一条 GitHub 安装命令即可：

```bash
npx skills add https://github.com/moyojo/skills --skill xiuzhen-art-generator --agent codex --yes
```

如果是在本仓库内开发或调试，也可以从本地目录安装当前工作区版本：

```bash
npx skills add . --skill xiuzhen-art-generator --agent codex --yes
```

## 使用

安装后，在 Codex 中直接提出修真/仙侠设定生成需求即可触发 skill。也可以显式点名 skill：

```text
$xiuzhen-art-generator 生成一个金丹境剑法，偏追索、破障和宗门秘传风格
```

下面展示几组 Codex 使用该 skill 生成后的结果。相同请求在不同会话中可能因为随机抽样与模型发挥而变化。

### 示例一：金丹境追索破障法术

```text
$xiuzhen-art-generator 生成一个金丹境法术，偏追索、破障和宗门秘传风格
```

生成结果：

```text
1. 设定总览

此术为金丹境宗门秘传法术，偏旁门丹言一脉，名列内府玉册，非真传长老不得观全诀。它不以蛮力轰碎阵禁，而是先追索目标在十日内留下的“将应之劫”，再以丹诀把障壁、迷阵、封印、遁光的闭合状态炼成可蜕之壳，从缝中开路、揭藏、逼形。

2. 核心三位

天位：丹、言。
地位：旬日为主承载，萌期为触发，朔望为锁定。
人位：化为主效用，劫期为索敌识别。

3. 天位动力因

丹

丹在此术中不是药物题材，而是“偏性被收束后必然转熟”的内炼压力。凡丹药入腹，药性不会停在原处，必沿脉象寻找可归、可化、可排之处；此术取的正是这种“未化尽而必求出路”的势，使障壁也被视作一枚未成之丹。

为何成力：施术者先服一丸宗门秘制的“沉脉旬砂”，其药性入丹田后不立刻发散，而在金丹外结成一层细密药壳。药壳遇到禁制、迷障、遁术残痕时，会把对方的闭合、遮蔽、隔断视为尚未转熟的药滓，从而产生逼其蜕变的内压。

何因增强：障碍越是新成、越是气机未稳、越是靠多层禁制强行缝合，丹压越盛；若目标在十日内服丹、疗伤、催阵、换脉、祭器，此术更容易借其药性余痕追上去。金丹修士自身丹火越纯、识得药性越细，越能把破障之力压成一线。

何时耗尽：药壳一旦完全转熟，或障碍已被炼开一层、失去“未成待化”的状态，丹压便迅速衰减。若十日已过，残痕冷尽，术中旬砂也会沉回脉底，只剩寻常探路之效。

失控何状：丹压过盛时，施术者会把自身经脉、护体法光甚至金丹外壳也误判为“障”，出现脉中自裂、眼前万物皆有药滓之感；严重者会呕出带符纹的黑砂，三日内无法服丹疗伤。

如何度量：以舌下丹腥、指尖砂纹、脉中沉响为准。丹腥越苦，说明障碍越厚；砂纹成环，说明可破；若砂纹散成灰点，则此障已无可化之壳，强行施展只会伤己。

言

言在此术中不是口令权限，而是“定方、定性、定去向”的符号压力。丹术最怕药性散乱，丹诀一出口，便等于替未熟之物写下归经和火候；被言辞钉住的状态，会被迫向某一种解释靠拢。

为何成力：施术者念出的不是普通咒语，而是宗门传下的策问式丹诀。每一句都如医案定证：此气何来、此障何藏、此路何闭。言辞一旦切中，混乱的禁制与残痕便不能继续含糊，必须显出一种可被炼化的病机。

何因增强：对方遮掩越多、阵纹越自相矛盾、遁术越依赖假象，言力越强；若施术者能连续三问皆中，第三问落下时，障壁会出现短暂“自答”，把真正的薄弱处吐出。

何时耗尽：同一处障碍不能反复受同一句丹诀逼问。真相被问出、路径被定下、或施术者词穷重复时，言力便散。若对方是无机死禁、无残念、无药性、无改易余地，言辞只能震出表层回音。

失控何状：言力失控时，丹诀会反噬为乱方。施术者会听见许多不属于自己的答语，把友方护阵、旧伤、未了誓言都误作障碍；严重时一句咒未完，口中先自生丹砂，封住舌根。

如何度量：看“问声回转”。第一问有回音，可追；第二问回音变窄，可破；第三问若回音中带金石摩擦声，说明已触及障核。若回音过甜，反是伪答，需立刻止术。

4. 复合结构与纯化取舍

此术以丹为核心纲领，以言为供能主源。丹负责把障碍压入“待熟待蜕”的状态，言负责替这种状态定名、定脉、定去向。二者不是并列：无丹则言只成问路小咒，无言则丹压乱冲，容易破错地方。

它没有追求元婴级的大范围破界，而是纯化为金丹斗法中极实用的三件事：追残痕、辨障核、开一线。被舍弃的是强行摧毁整座大阵的威势，换来的是隐秘、精准和可连续用于宗门追捕。

5. 地位媒介与施展/修炼方式

旬日为主承载。施术前须以十枚旬符压在袖内或丹匣中，每一符记一日气机。目标在十日内留下的药气、阵气、伤气、遁光余痕，都能被旬符暂时收束成可问之线。

萌期为触发。此术最擅长抓“刚起而未稳”的变化：新布的禁制、刚闭合的石门、才转移的洞府入口、刚服丹疗伤后压下的气息。施术者以一缕初梦似的神识探入，不碰实处，只碰势头初生处。

朔望为锁定。月相不直接供能，只作感知尺。下弦夜施展最稳，因为盈亏相交，善辨藏露；若在朔日施展，追索强而破障弱；望日施展，破障强而隐蔽差。

6. 人位用途与效果

主效用为化，具体是蜕变。它不把障碍炸开，而是迫使障碍从“封闭完整”蜕成“有壳有缝”的状态。阵门会露出一道旧修补痕，迷障会显出最早立障者的一口气，遁术会留下转向前的一瞬脉动。

索敌识别为劫期。此术追的不是目标本人，而是目标将临的小劫：伤势复发、丹毒上涌、护符将裂、阵门将换值、遁光将换气。凡金丹以下修士仓促逃遁，多半会留下这种避劫窗口；金丹同阶若处理不净，也会被此术咬住三到十息。

7. 限制、代价与反噬

此术必须有痕可问，无痕、无变、无待熟之处便难以起效。对纯粹以蛮力镇压的厚重大阵，它只能找修补缝，不能代替破阵重器。

每次施展至少耗一枚旬符和一口沉脉旬砂。连续三次之后，施术者脉象会变沉，服用疗伤丹药的效果减半。若强追元婴修士留下的假劫，丹言会被反牵，轻则十日耳中闻风如咒，重则金丹外壳生出寒碑纹，闭关数月方能磨去。

8. 境界适配与等阶差距

筑基修士即便得了残诀，也多半只能用作寻人问路，无法承受沉脉旬砂压住金丹外壳的步骤，更难以在三问之间稳住药性与言力。

金丹修士正适合此术：有内丹可承药壳，有神识可辨障核，也有足够寿元和资源修成旬符秘匣。元婴修士若用此术，会嫌其距离短、窗口窄，通常会把“旬日”上炼为“岁差”或“劫数”，另成高阶推演破界法。

9. 适配场景

宗门追捕叛徒时，长老可在丹房余灰、洞府门槛、传送阵裂纹处施术，追出对方十日内必经的一处换气点。斗法时，它可在敌方护体宝光刚修补的一瞬问出薄弱处，一指点去，不破全罩，只令宝光自行蜕壳，露出半息空门。

10. 诸名

1. 三问蜕障印：宗门内府玉册所载之名，短促可施，正合三问定障、逼其蜕壳之法。
2. 沉脉问路术：长老手札常用私称，重在服砂沉脉后追索残痕，语气较实。
3. 旬砂策障法：传功堂记录残篇时的题名，点出旬符与丹砂两重媒介。
4. 下弦开壳印：弟子与执事间的俗称，多指下弦夜破门追人的用法。

11. 设定集正文

三问蜕障印，金丹境丹言秘术，出自栖霞门内府玉册。施术者先含沉脉旬砂，以十枚旬符收十日残气，再于下弦月色中连问三句：问其气所从来，问其障所藏处，问其路所未闭。三问若中，袖中旬符自焦一角，指下丹砂成环，阵禁、迷障、遁光、石门皆会在最薄处生出一层灰白药壳。此壳非实物，乃闭合之势被丹诀逼熟后的残相，金丹修士只须一指点破，便可开半息至三息的缝。

此术最忌贪功。若障中无痕而强问，问声会倒灌入脉；若把友方护阵误作敌踪，旬砂会封舌，令施术者十日不得念诀。栖霞门历代只传执法、守丹、巡山三脉，凡弟子私录半句，按盗丹律废去一脉修为。
```

### 示例三：元婴期因果梦境法宝

```text
设计一件偏因果与梦境的元婴期法宝，给出名称、来历、使用代价和克制方式
```

生成结果：

```text
名称：坐忘续火镜

类型：元婴期法宝
基调：上古失传，因果入梦，换气定界。
核心三位：天位「威、间歇」；地位「燃熄、复」；人位「边界、虚」。

天位动力因：
- 威：此宝的威不是身份压服，而是“后果将落未落”时的雷威。目标在梦中越知道自己有旧因未清，越会在元婴换气时生出畏缩，镜中雷纹便越亮；若目标无亏欠、因果已偿或能坦然承受后果，雷威便耗尽；失控时会反照器主旧债，使器主也在梦中听见雷声；以镜背雷纹的明暗、梦中跪影是否颤动来度量。
- 间歇：成力于呼吸、入定、出窍、转念之间的空隙。元婴离体、梦境换幕、因果将接未接时，间歇最大；若目标以秘法保持一念不断、梦境不换、气息不歇，此力便耗尽；失控时会把器主自身念头切成断片，醒后记忆缺页；以镜面火光一熄一燃的间隔长短来度量。

地位媒介：
- 燃熄：镜心有一线续火，负责承载梦境开合；火明则入梦，火暗则封界。
- 复：以旧物、旧誓、旧伤或旧名锁定目标，使梦境不断复现其未了之因。

人位效果链：
- 边界：主效用是在梦中划出一重药界，使目标元婴不能随意远遁、夺舍或借梦换身。
- 虚：识别目标尚未显化的潜能和退路，把最可能逃出的空隙提前标出，再由药界封住。

来历：
```

## 资料与数据

- `skills/xiuzhen-art-generator/SKILL.md`：skill 的主说明、触发条件、工作流和输出约定。
- `skills/xiuzhen-art-generator/data/pools.md`：生成器实际加载的权威池数据，包含范畴、要素、细分、主题偏置、条件别名、复合父级模型和剑性分支模型。
- `skills/xiuzhen-art-generator/scripts/generate_prompt.py`：提示词生成入口。
- `skills/xiuzhen-art-generator/scripts/pool_parser.py`：解析 `data/pools.md` 的结构化文本。
- `skills/xiuzhen-art-generator/scripts/validate_pools.py`：检查池数据完整性和引用有效性。
- `skills/xiuzhen-art-generator/references/`：创作参考资料，包括三位机制、境界体系和命名风格。
- `docs/修真百艺.md`：面向人阅读的背景说明。

## 维护规则

新增或调整修真要素、细分、主题偏置、条件别名、复合父级模型、剑性分支模型时，优先修改 `skills/xiuzhen-art-generator/data/pools.md`，不要把同类数据散落到脚本里。修改后运行：

```bash
python3 skills/xiuzhen-art-generator/scripts/validate_pools.py
python3 $CODEX_HOME/skills/.system/skill-creator/scripts/quick_validate.py skills/xiuzhen-art-generator
```

## 开发约定

- 非代码内容默认使用简体中文，以保持修真/仙侠语境。
- Python 脚本面向 Python 3，保持无额外依赖，必要时使用类型提示。
- 需要可复现输出时使用 `--seed`。
- 修改生成逻辑后，至少测试随机路径、受限路径和 seeded 路径。
